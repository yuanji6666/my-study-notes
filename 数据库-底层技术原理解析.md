# 索引
## innoDB页结构

当从表中取出一行数据时，实际上取出一页（16kb）数据到内存中

**页是磁盘与内存之间交互的基本单位**

## innoDB行格式

区别于逻辑上的行，存储行时会加一些额外信息

compact行格式：

| 变长字段长度列表  | NULL列表          | 记录头信息          | 列1数据 | 列2数据 | 。。。 |
| --------- | --------------- | -------------- | ---- | ---- | --- |
| 记录varchar | 记录某列是否存了数据防浪费空间 | 若干信息包含指向下一行的指针 |      |      |     |

## innoDB聚集索引

可以观察到查询出来的数据是根据主键**自动排序**的

每行数据排列成一个**有序的**类似于链表的结构

链表太长，**页目录**会记录每组的位置，先查页目录确定它的大概位置
