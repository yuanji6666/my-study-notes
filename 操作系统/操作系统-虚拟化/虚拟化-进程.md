# 基础概念
## 程序与进程

程序是静态的代码和数据，进程就是进行起来的程序，进程有上下文容纳程序运行所需信息

每个进程独立分配了一块完整的和内存大小相等的进程地址空间

从进程视角来看，它独享了CPU和内存
## 逻辑流

一个进程程序计数器PC的一系列值构成**逻辑流**，可以看作程序运行流程；如果以时间为轴，流之间有重叠，它们就是**并发流**

## 用户模式/内核模式

本质是寄存器中的模式位控制当前进程的权限

内核模式里进程可以执行任何指令，访问所有内存；用户模式则不能

## 上下文切换

user model(A) -> process A blocking -> system call -> kernel model(store context A) -> kernel model(load context B) -> user model(B)
- across two processes in kernel model
- context switching only in kernel model (no Permisson in user model)
